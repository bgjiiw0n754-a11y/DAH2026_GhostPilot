Ghost Pilot SITL 피벗 과정 설명

오프라인 데모의 GLOBAL_POSITION_INT 공격이 왜 SITL에서는 달라졌고, 왜 GPS_INPUT direct-link PoC까지 확인했는지 팀 공유용으로 정리한 문서입니다.

핵심 한 줄

오프라인 가상환경에서는 GLOBAL_POSITION_INT를 직접 만들어 방어에 보여줄 수 있었지만, 실제 FC가 있는 SITL에서는 그 메시지가 항법 입력이 아니라 출력 상태 메시지라 반영되지 않았다. 그래서 FC가 실제로 받아들이는 GPS_INPUT 입력 계열로 피벗했고, direct-link 조건에서 14550 방어 입력 스트림까지 위치 변화가 도달하는 것을 확인했다.

보고서 표현 기준

오프라인: 공격/방어 로직 재현 SITL: FC 차이 확인 GPS_INPUT: direct-link PoC 확인 14550: 도달성 확인 Ghost 완전 성공 아님
주의: 이번 결과는 방어 end-to-end 탐지 검증이 아니라, 위조 위치가 방어 에이전트 입력 스트림까지 도달하는지 확인한 단계입니다.

포트와 역할 먼저 보기

5762 direct-link

FC에 직접 말하는 쪽입니다. 이번에 GPS_INPUT을 넣었을 때 실제 GPS fix, RAW GPS, EKF 위치가 반영된 경로입니다.

비유: 담당자 책상에 직접 서류를 올려두는 것.

14550 MAVProxy output

GCS나 방어 에이전트가 텔레메트리를 받아보는 출력 포트입니다. 이번에는 주입용이 아니라, 바뀐 위치가 방어 입력 스트림까지 나오는지 관찰하는 용도로 확인했습니다.

비유: 관제실 모니터가 실제로 어떤 화면을 받고 있는지 보는 것.

시도 과정

0

오프라인 데모에서는 왜 됐나

확인됨

오프라인/FakeMsg 데모에는 실제 FC가 없습니다. 방어 에이전트가 읽는 MAVLink 형태 입력 스트림을 우리가 직접 구성하므로 GLOBAL_POSITION_INT를 위조하면 방어가 그 값을 그대로 봅니다.

비유: 선생님에게 보여줄 답안지를 우리가 직접 만들어 주는 상황입니다. 답안지 숫자를 바꾸면 선생님은 바뀐 숫자를 봅니다.
1

SITL에서 GLOBAL_POSITION_INT 단독 위조 실패

문제 확인

SITL에는 ArduPilot FC가 있고, FC가 GPS/IMU/EKF를 통해 자기 위치를 계산한 뒤 GLOBAL_POSITION_INT를 출력합니다. 그래서 밖에서 같은 메시지를 보낸다고 FC 내부 위치가 바뀌지 않았습니다.

비유: 체온 모니터에 표시되는 숫자를 옆에서 따라 말한다고 사람 체온이 바뀌지는 않습니다. 출력값을 흉내 낸 것이지 센서 입력을 바꾼 것이 아닙니다.
2

입력 계열 후보로 GPS_INPUT 검토

피벗 후보

GLOBAL_POSITION_INT가 출력이라면, FC가 실제로 받아들이는 센서 입력 계열이 필요합니다. 그래서 외부 GPS 센서값처럼 넣을 수 있는 GPS_INPUT을 대체 엔진 후보로 보았습니다.

비유: 모니터 숫자를 바꾸는 대신, 체온계 센서가 읽은 값을 바꿔 넣는 방식으로 방향을 바꾼 것입니다.
3

초기 "움직임" 관찰은 오염으로 폐기

교란 발견

처음에는 GPS_INPUT을 넣었더니 위치가 움직인 것처럼 보였지만, 실제로는 이전에 띄운 fly_pattern 자율비행 프로세스가 살아 있었습니다. 따라서 그 관찰은 GPS_INPUT 효과로 볼 수 없어 폐기했습니다.

비유: 내가 리모컨을 눌러 TV 채널이 바뀐 줄 알았는데, 옆 사람이 다른 리모컨으로 채널을 돌리고 있었던 상황입니다.
4

MAVProxy UDP 경유 GPS_INPUT은 미반영

경로 문제

14551 같은 MAVProxy UDP output 쪽으로 GPS_INPUT을 넣었을 때는 FC의 GPS fix나 위치가 바뀌지 않았습니다. 이후 direct-link와 비교하면서 문제는 메시지 자체가 아니라 전달 경로였다는 쪽으로 좁혀졌습니다.

비유: 담당자에게 가야 하는 서류를 관람석 안내 데스크에 놓았는데, 그 데스크가 담당자에게 서류를 전달하지 않은 것입니다.
5

HIL_GPS와 SIM_GPS1_* 확인

분리 정리

HIL_GPS는 현재 펌웨어에서 지원 경로가 제거된 것으로 정리했습니다. SIM_GPS1_*는 SITL 내부 GPS 센서 모델을 직접 조작하면 반영되지만, 외부 공격 채널이 아니라 방어 검증용 테스트 도구로 분리했습니다.

비유: SIM_GPS1_*는 밖에서 드론을 속이는 게 아니라, 실험실 안의 가짜 GPS 센서 자체를 연구자가 손으로 조절하는 것에 가깝습니다.
6

GPS_INPUT direct-link 반영 확인

재현성 확인

GPS1_TYPE=14를 부팅 시 적용하고, FC direct-link 5762로 GPS_INPUT을 넣었을 때 fix가 1 -> 3, 위성 수가 0 -> 12로 바뀌고 RAW GPS/EKF 위치가 주입 목표값을 따라갔습니다. 반복 2회와 작은 offset에서도 재현되었습니다.

비유: 같은 서류를 이번에는 담당자 책상에 직접 올려두었더니 담당자가 바로 처리한 것입니다.
7

14550 방어 입력 스트림 도달성 확인

A5 핵심

주입은 direct-link 5762로 했고, 관찰은 MAVProxy output 14550에서 했습니다. 그 결과 GPS_RAW 위도와 GLOBAL_POSITION_INT 위도 변화가 14550에서도 관찰되었습니다. 이는 위조 위치가 방어 에이전트가 읽는 입력 스트림까지 도달했다는 의미입니다.

비유: 공격자가 CCTV에 가짜 영상을 넣었고, 그 가짜 영상이 관제실 모니터까지 실제로 도착했는지 확인한 것입니다. 관제원이 가짜라고 알아차렸는지는 아직 별도 문제입니다.

최종 체인

Step 1 GPS_INPUT 주입

5762 direct-link로 FC에 직접 도달

Step 2 GPS fix 회복

fix 1 -> 3, 위성 0 -> 12

Step 3 EKF 위치 반영

RAW GPS와 추정 위치가 목표 offset 추종

Step 4 GLOBAL_POSITION_INT 출력

FC가 바뀐 위치를 상태 메시지로 방송

Step 5 14550 도달

방어 에이전트 입력 스트림에서 위치 변화 관찰

채널별 판정표

경로 이번 판정 보고서 표현
GLOBAL_POSITION_INT SITL 항법 입력 미반영 오프라인에서는 재현 가능하지만, SITL에서는 출력 상태 메시지라 FC 내부 항법 상태를 바꾸지 못함.
GPS_INPUT via MAVProxy UDP 현재 경로 미전달 기본 MAVProxy UDP output 쪽 입력은 FC까지 역방향 전달되지 않음.
GPS_INPUT direct-link 항법 입력 반영 FC direct-link 조건에서 fix/RAW GPS/EKF 위치에 반영됨. 본선 엔진 피벗 후보.
14550 관찰 도달성 확인 direct-link 주입 결과가 방어 입력 스트림에서도 관찰됨. 단, 방어 탐지 검증은 아님.
HIL_GPS 현재 펌웨어 경로 제외 현 버전에서 처리 경로가 제거된 후보로 정리.
SIM_GPS1_* SITL 테스트 도구 반영은 되지만 외부 공격 채널이 아니라 내부 센서 모델 조작으로 분리.

이번에 말할 수 있는 것

  • 오프라인 데모는 예선 본체로 유지한다.
  • SITL에서는 FC 유무 때문에 GLOBAL_POSITION_INT 방식의 한계를 확인했다.
  • GPS_INPUT direct-link는 실제 항법 입력으로 반영된다.
  • 위치 변화가 14550 방어 입력 스트림까지 도달하는 것을 확인했다.
  • 이 케이스를 대표 사례로 삼아 본선에서 다른 공격면도 FC 입력/명령/텔레메트리 경로 기준으로 고도화한다.

아직 말하면 안 되는 것

  • SITL Ghost 공격 완전 성공
  • 실제 GPS spoofing 구현
  • 실제 드론 탈취
  • ghost_spoof 통합 완료
  • 방어 end-to-end 탐지 검증 완료

보고서용 문장 초안

예선 단계에서는 FakeMsg/합성 텔레메트리 기반 오프라인 환경에서 Ghost 공격과 방어 탐지 로직을 검증했다. 다만 실제 ArduPilot SITL에서는 FC가 존재하므로, 오프라인에서 재현된 GLOBAL_POSITION_INT 위조가 그대로 항법 상태에 반영되지 않는 한계를 확인했다. 이에 따라 FC가 실제로 수용하는 항법 입력 계열로 피벗했고, GPS_INPUT direct-link 조건에서 GPS fix, RAW GPS, EKF 위치 및 MAVProxy output 14550의 위치 변화가 관찰됨을 확인했다. 본 결과는 Ghost 공격 완성 검증이 아니라, 본선에서 ghost_spoof, 점진 편향, 적응형 백오프 로직을 실제 FC 입력 경로에 연결하기 위한 피벗 PoC로 분리한다.
